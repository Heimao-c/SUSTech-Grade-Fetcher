# SUSTech-Grade-Fetcher 成绩查询工具

一个基于 **CAS 登录** 的南方科技大学 **TIS 成绩查询工具**。支持自动获取 `route/JSESSIONID`，并按教务系统抓包流程请求数据，在终端格式化输出 **GPA 概览** 及 **详细成绩表（含班级排名）**。

本工具现已通过代码整合，适配 **本科生** 与 **研究生**。

---

## ✨ 功能特性

- **多身份适配**：一套代码同时支持本科生与研究生，运行后交互式选择身份即可。
- **班级排名显示**：对比教务系统移动端，本工具可直接查看每门课程的**班级排名**。
- **双版本选择**：
  - **Go 版本（推荐）**：单文件运行，支持交互式密码隐藏输入（输入时不显示字符），更安全便捷。
  - **Python 版本**：适合开发者二次开发或快速测试，支持 Python 3.8+ 环境。
- **保底方案**：提供 `cookie_version.py`，应对 CAS 登录触发验证码或二次认证导致自动化脚本失效的情况。
- **美化输出**：按学期分组展示成绩，支持中英文字符宽度自适应，表格对齐美观。

## 🚀 使用指南

### 1. Go 版本（推荐）
如果你不希望安装 Python 环境，可以直接使用编译好的程序。
1. **运行程序**：
   - Windows 用户可直接运行 `sustech-grade-fetcher.exe`。
   - 源码运行：`go run main.go`（需安装 Go 环境并保留 `go.mod`）。
2. **操作流程**：
   - 运行后根据提示输入 `1`（本科生）或 `2`（研究生）。
   - 输入学号。
   - 输入密码（**注意：输入密码时终端不会显示任何字符，输完直接按回车即可**）。

### 2. Python 版本
1. **安装依赖**：

   ```bash
   pip install -r requirements.txt
   ```


2. **运行脚本**：
    - 在脚本目录执行：
   ```bash
   python main.py
   ```


### 3.手动 Cookie 版（备选方案）
当 `main` 脚本因 CAS 验证码风控无法自动登录时：
1. 获取 Cookie：浏览器登录 TIS，按 F12 在“网络 (Network)”或“应用 (Application)”标签中找到 `route` 和 `JSESSIONID`。
2. 填写配置：用编辑器打开 `cookie_version.py`，将值填入顶部的 `TIS_ROUTE` 和 `TIS_JSESSIONID` 变量中。
3. 运行：`python cookie_version.py`。

### 🛠️ 常见问题 (FAQ)
1. **登录失败：触发验证码 / 二次认证**
   说明 CAS 开启了风控。请尝试切换网络环境（如从外网切回校园网）重试，或使用 `cookie_version.py` 手动提供 Cookie。
2. **成绩接口返回 500 或非 JSON**
   通常是由于教务系统接口变动或身份参数不匹配。本工具已针对本科生 (`01`) 与研究生 (`02`) 的不同请求路径进行了适配。
